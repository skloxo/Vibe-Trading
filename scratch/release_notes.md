# Vibe-Trading-CNX v0.1.10.cnx.1.3

本次更新专注于**局域网管理服务看板与实时运行日志**的深度集成，并对核心管理操作交互进行了全面的打磨与加固。

### 🌟 核心更新亮点

1. **设置页管理卡片彻底合并至服务看板**
   - 彻底将设置页（Settings）底部的“租户密钥管理 (Tenant API Keys)”和“系统版本管理”卡片，重组迁移至“服务看板 (`/monitor`)”页面。
   - 采用精致的**左右平衡对称式布局**：左侧展示租户密钥与隔离工作区管理表格，右侧上方展示行情网关状态连接池，右侧下方展示系统版本与一键升级卡片，实现一体化运维管理。
2. **密钥隐私安全性显著加固**
   - 移除了租户列表中以明文或长掩码显示 API 密钥的旧设计。
   - 代之以小巧精致、可防止旁视的“复制密钥”安全气泡按钮。用户点击后即可一键复制明文密钥并给出 Toast 成功气泡，在保护敏感密钥隐私的同时极大地提升了操作便利性。
   - 自助注册成功关闭模态框时，移除了对本地缓存的冗余密钥拉取副作用，使设置页轻量、干净。
3. **前端点击交互反馈升级（Toast 提示 + Loading 状态）**
   - 针对原“刷新状态”和“检查更新”按钮在点击时响应不明显、无任何成功/失败视觉状态指示的体验痛点，全面补齐了 Toast 动画气泡反馈。
   - 手动触发版本检查时，成功会提示“已是最新版本”或“发现新版本 X.Y.Z，请点击升级”；网关刷新成功时提示“行情网关状态刷新成功”，并在加载过程中展示旋转动画。
4. **日志页面独立诊断功能优化**
   - 保持在独立 `/logs` 路由下的“运行日志”终端的高性能重构，默认锁定过滤级别为 `WARNING` 以规避海量无用 INFO 渲染造成的客户端卡顿，且仅限局域网/本地回环模式展示。

---

### 📦 升级方法

**Docker / 一键更新**
在服务看板（`http://localhost:9888/monitor`）的右下角“系统版本管理”卡片中，点击“立即升级到 v0.1.10.cnx.1.3”，系统将自动触发平滑拉取并重启容器更新。
